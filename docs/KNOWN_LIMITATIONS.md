# Known Limitations

Tin Pata **v1.3.0** — personal-use, offline-first Android app.

---

## Platform & build

| Limitation | Detail |
|------------|--------|
| **Expo Go unsupported** | PDF reader and brightness require a **development or production build** (`npx expo run:android` or EAS APK). |
| **Android-only tested** | Primary target is Android. iOS may build but is not release-tested. |
| **Native folders not committed** | `android/` and `ios/` are generated by `expo prebuild` locally. See [CNG_SETUP.md](./CNG_SETUP.md). |
| **Personal use only** | No backend, no cloud sync, no multi-device library sync. |

---

## PDF reader

| Limitation | Detail |
|------------|--------|
| **Black flash on resume** | Opening at a saved page may briefly flash black (`react-native-pdf` native behavior). |
| **Fit/scroll apply at open only** | Fit mode and scroll mode are read once when the reader opens and frozen for that session. Changing settings while reading applies on the **next** open. |
| **Horizontal scroll varies by PDF** | Horizontal (page-by-page) mode uses native paging; some PDFs may layout or swipe differently. |
| **Focus mode is UI-only** | Focus mode hides app controls (toolbar, action bar) but does not modify the PDF renderer. |
| **No PDF text search** | Not planned for v1.3. |
| **Large PDFs** | Very large files may load slowly; progress depends on native renderer. |

Details: [READER_KNOWN_LIMITATIONS.md](./READER_KNOWN_LIMITATIONS.md)

---

## Data & backup

| Limitation | Detail |
|------------|--------|
| **PDFs not in JSON backup** | Export includes books metadata, sessions, notes, bookmarks, goals, reflections, and portable settings — **not PDF file bytes**. Copy PDFs separately. |
| **Restore requires local PDFs** | After import, books need matching PDF files at stored paths or re-import. |
| **No cloud backup** | Backup is manual JSON export via share sheet only. See [BACKUP_AND_RESTORE.md](./BACKUP_AND_RESTORE.md). |
| **Old backups compatible** | v1.2 adds `category` and `priority` on books; missing fields default on import. |

---

## Reader comfort

| Limitation | Detail |
|------------|--------|
| **Brightness needs native module** | `expo-brightness` must be in the built APK. Rebuild after adding the plugin. |
| **Brightness is system-level** | Affects device/app brightness, not PDF rendering internals. |
| **Keep awake is reader-only** | Screen stays on while the reader screen is open when enabled. |

---

## Stats & insights (v1.2+)

| Limitation | Detail |
|------------|--------|
| **Insights need session data** | Mood, focus, and blocker insights appear only when finish-session survey fields were saved. |
| **Estimated finish is approximate** | Based on average pages per reading day; needs at least 2 reading days and known total pages. |
| **No chart library** | Insights are card-based text summaries, not graphs. |
| **Time range buckets** | Best reading time uses fixed morning/afternoon/evening buckets, not exact hours. |

---

## If auto-resume regresses

If the initial `page` prop causes crashes again:

1. Revert to manual-only resume (remove reactive `page` prop).
2. Document on device in [READER_STABILITY_TEST_CHECKLIST.md](./READER_STABILITY_TEST_CHECKLIST.md).
3. Keep manual “Last read” fallback as the only resume path.
