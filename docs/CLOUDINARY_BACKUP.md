# Cloudinary Backup

Optional secure PDF backup for personal disaster recovery. **Not required for MVP** — implement in Phase 7.

---

## Design Goals

1. Never expose Cloudinary API secret in the mobile app
2. Keep local file as the reading source of truth
3. Store only `public_id` in SQLite — not permanent URLs
4. Use a minimal backend solely for signing upload/download requests
5. Backup is opt-in via Settings toggle

---

## Security Model

| Secret / Asset | Where It Lives |
|----------------|----------------|
| `CLOUDINARY_API_SECRET` | Backend `.env` only |
| `CLOUDINARY_API_KEY` | Backend `.env` only |
| `CLOUDINARY_CLOUD_NAME` | Backend `.env` + mobile app (public) |
| Signed upload signature | Generated by backend per request |
| Signed download URL | Generated by backend per request |
| `public_id` | SQLite after upload |
| Local PDF file | Device file storage |

### Why a Backend Is Needed

Cloudinary signed uploads require the API secret to generate a signature. Putting the secret in a mobile app means anyone can extract it from the APK/IPA. A minimal Express server holds the secret and returns short-lived signed parameters.

### MVP Without Backend

Phases 1–6 require **no backend**. The app works entirely on-device. Cloudinary integration is additive.

## Architecture

```
┌──────────────┐         ┌──────────────┐         ┌──────────────┐
│  Mobile App  │ ──────► │   Backend    │ ──────► │  Cloudinary  │
│              │ ◄────── │  (Express)   │         │              │
│ Local PDF    │         │ Signs only   │         │ Stores PDF   │
│ SQLite meta  │         │ No user DB   │         │ backup       │
└──────────────┘         └──────────────┘         └──────────────┘
```

Backend responsibilities **only**:
- `POST /api/cloudinary/sign-upload` → return signature, timestamp, api_key, folder
- `POST /api/cloudinary/sign-download` → return signed URL for a public_id
- `DELETE /api/cloudinary/delete` → delete asset by public_id (optional)

Backend does **not**:
- Store user accounts
- Store PDFs
- Proxy file uploads (app uploads directly to Cloudinary)

## Upload Flow

```
User imports PDF
        ↓
App saves PDF locally (always — local-first)
        ↓
If cloud_backup_enabled in settings:
        ↓
App → POST /api/cloudinary/sign-upload
      Body: { bookId, fileName }
        ↓
Backend generates signed upload params:
      { signature, timestamp, api_key, cloud_name, folder, public_id }
        ↓
App → POST https://api.cloudinary.com/v1_1/{cloud_name}/raw/upload
      (multipart form with file + signed params)
        ↓
Cloudinary returns { public_id, asset_id, ... }
        ↓
App saves to SQLite:
      books.cloudinary_public_id = public_id
      books.cloudinary_asset_id = asset_id
      books.is_uploaded = 1
```

### Upload Details

| Parameter | Value |
|-----------|-------|
| Resource type | `raw` (PDFs are raw files, not images) |
| Folder | `reading-habit/{user-device-id}/` or flat `reading-habit/` |
| `public_id` | Suggest `{bookId}` for predictable restore |
| Access mode | `authenticated` or signed URLs for download |

### Failure Handling

- Upload failure does not block import or reading
- `is_uploaded` remains 0
- Settings or book detail can show "Backup pending" with retry button

## Download / Restore Flow

```
App opens book but local file missing
(local_uri path does not exist)
        ↓
Check books.cloudinary_public_id
        ↓
If null → show "Re-import PDF" (no cloud copy)
        ↓
If present:
        ↓
App → POST /api/cloudinary/sign-download
      Body: { publicId }
        ↓
Backend returns { signedUrl, expiresAt }
        ↓
App downloads PDF from signedUrl
        ↓
App saves to documentDirectory/pdfs/{bookId}.pdf
        ↓
App updates SQLite:
      local_uri = new path
      is_downloaded = 1
        ↓
App opens local file (never stream from URL in reader)
```

### Restore Rules

- Always download to local storage first, then open local file
- Never point PDF viewer directly at Cloudinary URL
- Signed URLs are short-lived (e.g. 1 hour)

---

## Delete Flow

When user deletes a book from library:

```
1. Delete local PDF file
2. Delete SQLite records (cascade)
3. If cloudinary_public_id exists AND cloud_backup_enabled:
   → POST /api/cloudinary/delete { publicId }
   → Backend calls Cloudinary Admin API to destroy asset
```

If delete remote fails, local delete still succeeds.

## Backend API Specification

### Environment Variables

```env
CLOUDINARY_CLOUD_NAME=your_cloud_name
CLOUDINARY_API_KEY=your_api_key
CLOUDINARY_API_SECRET=your_api_secret
PORT=3000
# Optional: simple API key to prevent public abuse
APP_API_KEY=random_long_string
```

### POST `/api/cloudinary/sign-upload`

**Request:**
```json
{
  "bookId": "uuid",
  "fileName": "my-book.pdf"
}
```

**Response:**
```json
{
  "cloudName": "xxx",
  "apiKey": "xxx",
  "timestamp": 1718280000,
  "signature": "abc123...",
  "folder": "reading-habit",
  "publicId": "reading-habit/uuid",
  "resourceType": "raw"
}
```

### POST `/api/cloudinary/sign-download`

**Request:**
```json
{
  "publicId": "reading-habit/uuid"
}
```

**Response:**
```json
{
  "signedUrl": "https://res.cloudinary.com/...",
  "expiresAt": "2026-06-13T15:00:00Z"
}
```

### DELETE `/api/cloudinary/delete`

**Request:**
```json
{
  "publicId": "reading-habit/uuid"
}
```

**Response:**
```json
{
  "success": true
}
```

### Security Hardening (Recommended)

| Measure | Purpose |
|---------|---------|
| `APP_API_KEY` header check | Prevent random internet abuse of your signer |
| Rate limiting | Limit sign requests per IP |
| CORS | Allow only your app if needed |
| No auth in MVP | Acceptable for personal use with API key |

```typescript
// Phase 7 — interface sketch
interface CloudinaryService {
  uploadPdf(bookId: string): Promise<void>;
  restorePdf(bookId: string): Promise<string>; // returns new local_uri
  deleteRemotePdf(publicId: string): Promise<void>;
  isBackupEnabled(): boolean;
}
```

### Settings Integration

| Setting | Default | Effect |
|---------|---------|--------|
| `cloud_backup_enabled` | `false` | When true, auto-upload after import |
| `backend_api_url` | empty | Base URL for signer (e.g. `https://my-api.railway.app`) |
| `backend_api_key` | empty | Optional `APP_API_KEY` header value |

Store `backend_api_url` in settings so you can change deployment without app rebuild (or use app config for simplicity).

---

## What the Mobile App Must Never Do

- ❌ Store `CLOUDINARY_API_SECRET`
- ❌ Generate upload signatures client-side
- ❌ Store permanent public Cloudinary URLs in SQLite
- ❌ Use Cloudinary as primary PDF source for reading
- ❌ Require network to read a locally imported book

---

## Deployment Notes (Personal Use)

- Deploy backend to Railway, Render, or Fly.io (free tiers sufficient)
- One Cloudinary free account is enough for personal PDF backup
- No database needed on backend
- Estimated backend code: ~100–150 lines of Express
